## ✅ Features a Implementar

### 🤖 1. Insights Inteligentes com IA

**Problema:** Dashboard mostra apenas dados brutos (KPIs, gráficos), sem análise ou interpretação.

**Solução:** Adicionar camada de análise automática que gera mensagens personalizadas.

#### 📋 Checklist:

**Backend:**
- [ ] Criar função `generate_insights(user_id, period)`:
  - Comparar stats do período atual vs anterior
  - Calcular % de evolução (pace, distância, frequência)
  - Detectar recordes pessoais (PR)
  - Identificar padrões (dias preferidos, horários)
- [ ] Endpoint `GET /api/stats/insights?period=week`:
  ```json
  {
    "insights": [
      {
        "type": "improvement",
        "metric": "pace",
        "message": "Seu pace melhorou 8% essa semana! 🚀",
        "icon": "trending-up"
      },
      {
        "type": "pr",
        "metric": "distance",
        "message": "Novo recorde: 45 km em uma semana! 🏆",
        "icon": "trophy"
      },
      {
        "type": "consistency",
        "message": "Você correu em 4 dias diferentes. Excelente consistência!",
        "icon": "check-circle"
      }
    ]
  }
  ```

**Frontend:**
- [ ] Criar componente `InsightsCard.tsx`:
  - Lista de insights com ícones coloridos
  - Badges por tipo (melhoria/recorde/alerta)
  - Animação de entrada (fade-in)
- [ ] Adicionar card no dashboard (após KPIs)
- [ ] Tooltip com explicação do cálculo

#### 🎨 UI Mockup:
```
┌────────────────────────────────────┐
│  💡 Insights da Semana             │
├────────────────────────────────────┤
│  🚀 Seu pace melhorou 8%           │
│     De 06:15/km para 05:45/km      │
│                                    │
│  🏆 Novo recorde: 45 km/semana     │
│     Anterior: 38 km                │
│                                    │
│  ✅ 4 dias de treino               │
│     Consistência excelente!        │
└────────────────────────────────────┘
```

#### Tipos de Insights:
- **Melhoria:** pace, distância, frequência
- **Recorde:** maior distância semanal, melhor pace
- **Alerta:** queda de performance, semanas sem treinar
- **Padrão:** dias preferidos, horários consistentes
- **Meta:** progresso em relação à próxima prova

**Problema:** Provas cadastradas não influenciam o dashboard (sem relação com treinos).

**Solução:** Mostrar progresso em direção à meta da próxima prova.

#### 📋 Checklist:

**Backend:**
- [ ] Endpoint `GET /api/races/:id/progress`:

## 🗄️ Alterações no Banco de Dados

### Migration: `v1.2.0-insights-receipts.sql`

```sql
-- 1. Adicionar campo de comprovante nas provas
ALTER TABLE choppinho.race_registrations
ADD COLUMN IF NOT EXISTS receipt_url TEXT;

-- 2. Tabela de insights gerados (opcional, para histórico)
CREATE TABLE IF NOT EXISTS choppinho.insights (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL REFERENCES choppinho.users(id) ON DELETE CASCADE,
  period_start DATE NOT NULL,
  period_end DATE NOT NULL,
  insight_type VARCHAR(50) NOT NULL, -- 'improvement', 'pr', 'alert', 'pattern'
  metric VARCHAR(50), -- 'pace', 'distance', 'frequency'
  message TEXT NOT NULL,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_insights_user_date
ON choppinho.insights (user_id, period_start DESC);
```

## 🐛 Riscos e Mitigações

| Risco | Impacto | Mitigação |
|-------|---------|-----------|
| Insights incorretos causam confusão | Alto | Testes com dados reais, revisão de lógica |
| Upload de arquivo malicioso | Médio | Validação de tipo MIME, antivírus no storage |
| Performance ruim com muitos filtros | Baixo | Índices no banco, cache de queries |
