Evolution API
Ponte com WhatsApp
Conecta seu WhatsApp ao ecossistema. Recebe e envia mensagens via API REST.
Papel: Transporte de mensagens. Faz a ponte entre WhatsApp ↔ Chatwoot e WhatsApp ↔ Typebot.
Chatwoot
Inbox & Histórico
Central de conversas. Armazena histórico, permite atendimento humano e mantém o contexto do contato.
Papel: CRM de conversas. Fonte da verdade do contato + permite takeover humano quando o bot não resolve.
N8N
Orquestrador
O cérebro da operação. Recebe webhooks, decide o que fazer, roteia entre Typebot/Chatwoot, grava no banco e dispara ações.
Papel: Lógica de negócio. Decide se inicia onboarding, faz follow-up, processa callback do Strava, gera insights.
Typebot
Fluxo conversacional
Cuida da conversa estruturada: coleta nome, preferências e confirma ações. UI de arrastar e soltar.
Papel: Interface amigável de coleta de dados. O N8N inicia e encerra sessões do Typebot conforme necessário.
Strava OAuth + Webhooks
Dados de atividades
Autorização OAuth2 pra ler atividades. Webhook push pra notificar novas corridas em tempo real.
Papel: Fonte dos dados de treino. O N8N processa os callbacks e webhooks do Strava.
PostgreSQL
Base de dados
Armazena users, tokens, atividades e preferências. O N8N lê e grava diretamente via nó Postgres.
Papel: Persistência. Tudo que precisa sobreviver entre sessões fica aqui.
💡 Quem fala com quem?

• Evolution API → integra com Chatwoot (inbox nativo) e com Typebot (via integração direta ou N8N)
• Chatwoot → envia webhooks pro N8N a cada mensagem recebida
• N8N → orquestra tudo: chama o Typebot via API, grava no Postgres, envia mensagens via Evolution API, processa callbacks do Strava
• Typebot → coleta dados e devolve pro N8N via webhook ao finalizar
Fluxo de Onboarding Completo
Evolution API Chatwoot
1. Usuário manda "OI" no WhatsApp
Mensagem chega na Evolution API → cria/atualiza contato no Chatwoot → dispara webhook pro N8N
A Evolution API já cria o contato no Chatwoot automaticamente com phone_number e name do perfil do WhatsApp.
O Chatwoot dispara um webhook message_created pro N8N.
N8N Postgres
2. N8N recebe webhook e decide o fluxo
Consulta o banco por phone_number. Novo usuário? Inicia onboarding. Já existe? Redireciona conforme status.
Lógica de roteamento no N8N:
• Usuário novo → cria registro no Postgres com status = started → inicia Typebot
• Status started ou collecting_info → retoma Typebot
• Status awaiting_strava → reenvia link OAuth
• Status completed → responde normalmente (ou passa pro Chatwoot humano)
Typebot Evolution API
3. Typebot coleta dados básicos
Fluxo conversacional via WhatsApp — nome, confirmação de interesse e envio do link do Strava
O Typebot envia mensagens via integração com Evolution API:

→ "Como posso te chamar?" → salva first_name
→ "Qual seu e-mail?" → (opcional) salva email
→ "Vou te mandar o link pra conectar seu Strava. É rapidinho!"

Ao finalizar, o Typebot chama um webhook do N8N com os dados coletados.
↓ Typebot envia webhook pro N8N com dados coletados
N8N Postgres Evolution API
4. N8N salva dados e envia link do Strava
Grava nome/email no Postgres, gera URL OAuth2 personalizada e envia pelo WhatsApp via Evolution API
O N8N gera a URL com o state vinculado ao user_id:

https://strava.com/oauth/authorize?client_id=XXX&redirect_uri=https://seu-dominio.com/strava/callback&state={user_id}&scope=read,activity:read_all

Atualiza onboarding_status → awaiting_strava
Envia a mensagem pelo endpoint POST /message/sendText da Evolution API.
Strava N8N Postgres
5. Callback do OAuth — Strava autorizado
Usuário clica, autoriza, Strava redireciona com code → N8N troca por tokens e salva
O callback chega num webhook do N8N (ou endpoint seu):

1. Recebe code + state (user_id)
2. Troca o code por access_token + refresh_token via POST https://www.strava.com/oauth/token
3. Salva tokens + strava_athlete_id na tabela strava_connections
4. Atualiza onboarding_status → strava_connected
↓ N8N confirma conexão e inicia coleta de preferências
N8N Typebot Evolution API
6. Preferências de notificação
N8N dispara segundo fluxo do Typebot para configurar notificações
Após confirmar conexão, o N8N envia via Evolution API:
→ "Strava conectado com sucesso! 🎉"

E inicia novo fluxo do Typebot:
→ "Quer receber insights das suas corridas?" → notify_run_insights
→ "Resumo semanal de treinos?" → notify_weekly_summary
→ "Alertas de metas e recordes?" → notify_goals
→ "Dicas de treino?" → notify_training_tips

Typebot envia os resultados pro N8N via webhook ao finalizar.
N8N Postgres Evolution API Chatwoot
7. Onboarding completo ✓
Salva preferências, marca onboarding como concluído, envia boas-vindas final e atualiza Chatwoot
O N8N executa tudo em sequência:

1. Salva preferências na tabela notification_preferences
2. Atualiza onboarding_status → completed + onboarding_completed_at
3. Envia mensagem final via Evolution API: "Tudo pronto! Agora é só correr que eu cuido do resto 🏃"
4. Adiciona label onboarded no contato do Chatwoot (pra filtros e relatórios)
5. Registra webhook do Strava pra receber novas atividades
N8N Strava Postgres
8. (Bônus) Sync inicial de atividades
Puxa as últimas N atividades do Strava pra já ter dados no banco e gerar um primeiro insight
Opcional mas recomendado — dá uma boa primeira impressão:

1. GET /api/v3/athlete/activities?per_page=10
2. Salva na tabela activities
3. Gera um insight de boas-vindas: "Vi que sua última corrida foi de 5km em 28min. Tá indo bem! 💪"
💡 Follow-up automático: Crie um workflow no N8N com trigger de Schedule (diário) que busca users com onboarding_status ≠ completed há mais de 24h e envia lembrete via Evolution API.
💡 Typebot vs N8N — quem faz o quê?

• Typebot cuida da conversa (perguntas, validações, UX amigável)
• N8N cuida da lógica (roteamento, gravação no banco, chamadas de API, decisões)

Não misture: evite colocar lógica complexa no Typebot ou conversa no N8N.

Webhooks & Integrações
Chatwoot → N8N
Toda mensagem nova no WhatsApp chega aqui
// Configurar em: Chatwoot → Settings → Integrations → Webhooks
POST https://seu-n8n.com/webhook/chatwoot-incoming

// Eventos recomendados:
message_created
conversation_status_changed
contact_created
Dica: No N8N, filtre pelo event e message_type = incoming pra ignorar mensagens enviadas pelo próprio bot.
Typebot → N8N
Dados coletados no fluxo conversacional
// No Typebot: use o bloco "Webhook" no final do fluxo
POST https://seu-n8n.com/webhook/typebot-onboarding-complete

// Payload exemplo:
{
  "phone": "+5511999999999",
  "first_name": "João",
  "email": "joao@email.com",
  "wants_strava": true
}
Strava OAuth → N8N
Callback após o usuário autorizar
// Configurar como redirect_uri no Strava App
GET https://seu-n8n.com/webhook/strava-callback?code=XXX&state={user_id}

// N8N então faz POST pra trocar o code:
POST https://www.strava.com/oauth/token
{
  "client_id": "...",
  "client_secret": "...",
  "code": "XXX",
  "grant_type": "authorization_code"
}
⚠️ Importante: Após trocar o code, redirecione o usuário pra uma página de sucesso (pode ser uma página simples dizendo "Pode voltar pro WhatsApp!"). O N8N cuida de avisar o usuário pelo WhatsApp automaticamente.
Strava Push → N8N
Webhook de novas atividades (pós-onboarding)
// Registrar uma única vez via Strava API
POST https://www.strava.com/api/v3/push_subscriptions
{
  "callback_url": "https://seu-n8n.com/webhook/strava-activity",
  "verify_token": "seu-token-secreto"
}

// Strava manda a cada atividade nova:
{
  "object_type": "activity",
  "aspect_type": "create",
  "object_id": 12345678,
  "owner_id": 98765 // strava_athlete_id
}
Fluxo no N8N: Recebe push → busca owner_id na tabela strava_connections → puxa detalhes da atividade via API → salva em activities → gera insight → envia pelo WhatsApp via Evolution API.
N8N → Evolution API
Envio de mensagens para o usuário
// Para enviar mensagens fora do Typebot (insights, follow-ups)
POST https://sua-evolution.com/message/sendText/{instance}
{
  "number": "+5511999999999",
  "text": "Sua corrida de hoje foi incrível! 🏃 5.2km em 26min..."
}
Resumo dos Workflows N8N
Os 6 workflows que você vai precisar criar
1. Onboarding Router Trigger: webhook Chatwoot → decide fluxo baseado no status do user
2. Typebot Data Handler Trigger: webhook Typebot → salva dados → envia link Strava
3. Strava OAuth Callback Trigger: GET webhook → troca code por tokens → salva → notifica
4. Strava Activity Processor Trigger: webhook push → busca detalhes → salva → gera insight → envia
5. Follow-up Scheduler Trigger: Schedule (diário) → lembra users com onboarding incompleto
6. Weekly Summary Generator Trigger: Schedule (semanal) → agrega atividades → gera resumo → envia
